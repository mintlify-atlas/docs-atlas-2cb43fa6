---
title: "Solve arithmetic with a seq2seq model in neu"
description: "Train encoder-decoder seq2seq models to solve addition problems as character sequences using neu's Seq2Seq, PeekySeq2Seq, and AttentionSeq2Seq in Go."
---

Sequence-to-sequence (seq2seq) models map an input sequence to an output sequence of a different length. neu ships three variants — `Seq2Seq`, `PeekySeq2Seq`, and `AttentionSeq2Seq` — and includes a built-in arithmetic dataset where models learn to add numbers represented as character sequences (for example, `"71 + 9  "` → `"80 "`). This guide shows how to download the data, train each variant, and interpret the results.

## Prerequisites

- Go 1.22 or later
- Clone the repository: `git clone https://github.com/itsubaki/neu.git && cd neu`

## Download the arithmetic dataset

The addition dataset contains 50,000 character-sequence addition problems. Download it with:

```bash
make additiondl
# curl -fs -o testdata/addition.txt https://...
```

`sequence.Load` reads the file, builds a character vocabulary, and returns train/test splits as integer ID sequences:

```go
x, t, v := sequence.Must(sequence.Load(dir, sequence.AdditionTxt))
// x.Train / x.Test  [][]int — input sequences (the question side)
// t.Train / t.Test  [][]int — label sequences (the answer side)
// v                 *sequence.Vocab — rune ↔ ID mappings
```

Each input is a fixed-width string padded with spaces, e.g. `"7 + 906 "` (7 characters). Each label starts with a delimiter token `_` followed by the digit characters of the sum.

## Encoder-decoder flow

All three variants share the same high-level forward pass:

1. **Encoder** reads the reversed input sequence one time-step at a time, compressing it into a single context vector `h`.
2. **Decoder** receives `h` as its initial hidden state and generates the output sequence one character at a time, conditioned on the previous output.
3. **TimeSoftmaxWithLoss** computes the cross-entropy loss over all decoder output time steps.

```go
// inside Seq2Seq.Forward:
h     := m.Encoder.Forward(xs)         // xs: (T, N, 1) → h: (N, H)
score := m.Decoder.Forward(dxs, h)     // dxs: teacher-forced decoder input
loss  := m.Softmax.Forward(score, dts) // dts: target token IDs
```

The trainer reverses the input sequence before feeding it to the encoder — `vector.Reverse(Time(xs[begin:end]))` — which empirically improves convergence on short sequences.

## Model variants

<Tabs>
  <Tab title="Seq2Seq (basic)">
    `NewSeq2Seq` pairs a standard `Encoder` with a standard `Decoder`. The context vector `h` is only passed as the initial hidden state of the decoder LSTM.

    ```go
    m := model.NewSeq2Seq(&model.RNNLMConfig{
        VocabSize:   len(v.RuneToID), // 13 distinct characters
        WordVecSize: wordvecSize,     // default 64
        HiddenSize:  hiddenSize,      // default 128
        WeightInit:  weight.Xavier,
    })
    ```

    Architecture summary:

    ```
    *model.Seq2Seq
     0: *model.Encoder
     1:   *layer.TimeEmbedding: W(13, 64)
     2:   *layer.TimeLSTM: Wx(64, 512), Wh(128, 512), B(1, 512)
     3: *model.Decoder
     4:   *layer.TimeEmbedding: W(13, 64)
     5:   *layer.TimeLSTM: Wx(64, 512), Wh(128, 512), B(1, 512)
     6:   *layer.TimeAffine: W(128, 13), B(1, 13)
     7: *layer.TimeSoftmaxWithLoss
    ```
  </Tab>

  <Tab title="PeekySeq2Seq">
    `NewPeekySeq2Seq` uses a `PeekyDecoder` that concatenates the context vector `h` to the input of the decoder LSTM at **every** time step — not just the initial hidden state. This gives the decoder constant access to the full encoded representation, which accelerates learning.

    ```go
    m := model.NewPeekySeq2Seq(&model.RNNLMConfig{
        VocabSize:   len(v.RuneToID),
        WordVecSize: wordvecSize, // default 64
        HiddenSize:  hiddenSize,  // default 128
        WeightInit:  weight.Xavier,
    })
    ```

    Because `h` is concatenated to the embedding at each step, the decoder LSTM input width grows from `D` to `D + H`, and the affine output layer input width grows from `H` to `H + H`:

    ```
    *model.PeekySeq2Seq
     4:   *layer.TimeLSTM: Wx(192, 512), Wh(128, 512), B(1, 512)
     6:   *layer.TimeAffine: W(256, 13), B(1, 13)
    ```

    This is the variant used in `cmd/seq2seq/main.go`.
  </Tab>

  <Tab title="AttentionSeq2Seq">
    `NewAttentionSeq2Seq` replaces the fixed context vector with an attention mechanism. The encoder now outputs **all** hidden states (one per input time step), and the decoder uses a `TimeAttention` layer to compute a weighted sum over those states at each decode step.

    ```go
    m := model.NewAttentionSeq2Seq(&model.RNNLMConfig{
        VocabSize:   len(v.RuneToID),
        WordVecSize: wordvecSize, // default 64
        HiddenSize:  hiddenSize,  // default 128
        WeightInit:  weight.Xavier,
    })
    ```

    The attention variant is used in `cmd/seq2seq_attention/main.go` on the more challenging **date conversion** dataset (`sequence.DateTxt`), where input and output lengths differ significantly.
  </Tab>
</Tabs>

## Training

All variants use the same `trainer.NewSeq2Seq` and `Seq2SeqInput`. The Adam optimizer with gradient clipping works well across all three:

```go
tr := trainer.NewSeq2Seq(m, &optimizer.Adam{
    Alpha: alpha, // default 0.001
    Beta1: beta1, // default 0.9
    Beta2: beta2, // default 0.999
    Hooks: []optimizer.Hook{
        hook.GradsClipping(max), // default 5.0
    },
})

tr.Fit(&trainer.Seq2SeqInput{
    Train:      xt,
    TrainLabel: tt,
    Epochs:     epochs,    // default 100
    BatchSize:  batchSize, // default 2
    Verbose: func(epoch, j int, loss float64, m trainer.Seq2Seq) {
        if epoch%20 != 0 || j != 0 {
            return
        }
        tacc := generate(xt, tt, m, v, 5)
        vacc := generate(xv, tv, m, v, 5)
        fmt.Printf("%2d, %2d: loss=%.04f, train_acc=%.4f, test_acc=%.4f\n",
            epoch, j, loss, tacc, vacc)
    },
})
```

At inference time, `m.Generate` runs the encoder once and then auto-regressively decodes:

```go
tq    := vector.Reverse(trainer.Time(matrix.New(q))) // reverse input
guess := m.Generate(tq, correct[0], len(correct[1:]))
```

## Complete example

```go
package main

import (
	"flag"
	"fmt"
	"time"

	"github.com/itsubaki/neu/dataset/sequence"
	"github.com/itsubaki/neu/math/matrix"
	"github.com/itsubaki/neu/math/vector"
	"github.com/itsubaki/neu/model"
	"github.com/itsubaki/neu/optimizer"
	"github.com/itsubaki/neu/optimizer/hook"
	"github.com/itsubaki/neu/trainer"
	"github.com/itsubaki/neu/weight"
)

func main() {
	var dir string
	var epochs, dataSize, wordvecSize, hiddenSize, batchSize int
	var alpha, beta1, beta2, max float64
	flag.StringVar(&dir, "dir", "./testdata", "")
	flag.IntVar(&epochs, "epochs", 100, "")
	flag.IntVar(&dataSize, "data-size", -1, "")
	flag.IntVar(&wordvecSize, "wordvec-size", 64, "")
	flag.IntVar(&hiddenSize, "hidden-size", 128, "")
	flag.IntVar(&batchSize, "batch-size", 2, "")
	flag.Float64Var(&alpha, "alpha", 0.001, "")
	flag.Float64Var(&beta1, "beta1", 0.9, "")
	flag.Float64Var(&beta2, "beta2", 0.999, "")
	flag.Float64Var(&max, "grads-cliping-max", 5.0, "")
	flag.Parse()

	x, t, v := sequence.Must(sequence.Load(dir, sequence.AdditionTxt))
	xt, tt := x.Train, t.Train
	xv, tv := x.Test, t.Test
	if dataSize > 0 {
		xt, tt = x.Train[:dataSize], t.Train[:dataSize]
		xv, tv = x.Test[:dataSize], t.Test[:dataSize]
	}

	m := model.NewPeekySeq2Seq(&model.RNNLMConfig{
		VocabSize:   len(v.RuneToID),
		WordVecSize: wordvecSize,
		HiddenSize:  hiddenSize,
		WeightInit:  weight.Xavier,
	})

	fmt.Println(m.Summary()[0])
	for i, s := range m.Summary()[1:] {
		fmt.Printf("%2d: %v\n", i, s)
	}
	fmt.Println()

	tr := trainer.NewSeq2Seq(m, &optimizer.Adam{
		Alpha: alpha,
		Beta1: beta1,
		Beta2: beta2,
		Hooks: []optimizer.Hook{
			hook.GradsClipping(max),
		},
	})

	now := time.Now()
	tr.Fit(&trainer.Seq2SeqInput{
		Train:      xt,
		TrainLabel: tt,
		Epochs:     epochs,
		BatchSize:  batchSize,
		Verbose: func(epoch, j int, loss float64, m trainer.Seq2Seq) {
			if epoch%20 != 0 || j != 0 {
				return
			}
			tacc := generate(xt, tt, m, v, 5)
			vacc := generate(xv, tv, m, v, 5)
			fmt.Printf("%2d, %2d: loss=%.04f, train_acc=%.4f, test_acc=%.4f\n",
				epoch, j, loss, tacc, vacc)
			fmt.Println()
		},
	})

	fmt.Printf("elapsed=%v\n", time.Since(now))
}

func generate(x, t [][]int, m trainer.Seq2Seq, v *sequence.Vocab, top int) float64 {
	xs, ts := vector.Shuffle(x, t)
	xm := matrix.From(xs)

	var acc int
	for k := range top {
		q, correct := xm[k], ts[k]
		tq := vector.Reverse(trainer.Time(matrix.New(q)))

		guess := m.Generate(tq, correct[0], len(correct[1:]))
		if vector.Equal(correct[1:], guess) {
			acc++
		}

		fmt.Printf("%v %v; %v\n", v.ToString(xs[k]), v.ToString(correct), v.ToString(guess))
	}

	return float64(acc) / float64(top)
}
```

## Running the examples

```bash
# PeekySeq2Seq on addition (quick smoke test, 10 samples)
make seq2seq
# go run cmd/seq2seq/main.go --dir ./testdata --data-size 10

# AttentionSeq2Seq on date conversion
make seq2seq_attention
# go run cmd/seq2seq_attention/main.go --dir ./testdata --data-size 10
```

## Expected output

The format is `[input] [correct]; [predicted]`. Early epochs produce random guesses; by epoch 40 the model solves most problems in the training set:

```
# epoch 20 — still learning
[7 + 9 0 6    ] [_ 9 1 3  ]; [9 9 1  ]
[5 4 4 + 4 2  ] [_ 5 8 6  ]; [2 1    ]
20,  0: loss=0.8230, train_acc=0.0000, test_acc=0.0000

# epoch 40 — training set solved, generalisation beginning
[1 9 4 + 7    ] [_ 2 0 1  ]; [2 0 1  ]
[5 4 4 + 4 2  ] [_ 5 8 6  ]; [5 8 6  ]
[7 8 8 + 2 2 7] [_ 1 0 1 5]; [1 0 1 5]
40,  0: loss=0.0912, train_acc=1.0000, test_acc=0.0000
```

<Tip>
  For the attention variant, download the date-conversion dataset with `make datedl` and swap `sequence.AdditionTxt` for `sequence.DateTxt`. The date dataset has longer, variable-length sequences where attention provides a more visible accuracy advantage.
</Tip>
