neu ships three dataset loaders that cover the most common introductory experiments: image classification with MNIST, language modeling with the Penn Treebank corpus, and sequence-to-sequence learning with synthetic arithmetic problems. All loaders read from the local filesystem; data files must be downloaded separately before use. The `testdata/` directory at the repository root is the conventional cache location used by the project's own test suite and `make` targets.

<Tab title="MNIST">

## MNIST

MNIST is a dataset of 70,000 grayscale images of handwritten digits (0–9), split into 60,000 training samples and 10,000 test samples. Each image is 28×28 pixels stored as raw bytes.

**Constants**

| Constant | Value | Description |
|---|---|---|
| `mnist.Width` | `28` | Image width in pixels. |
| `mnist.Height` | `28` | Image height in pixels. |
| `mnist.Labels` | `10` | Number of distinct digit classes (0–9). |

**Types**

```go
type Image [Width * Height]byte  // 784-byte flat pixel array
type Label uint8                 // digit class 0–9

type Dataset struct {
    N     int     // number of samples
    Image []Image
    Label []Label
}
```

**Functions**

| Function | Signature | Description |
|---|---|---|
| `Load` | `func Load(dir string) (*Dataset, *Dataset, error)` | Load training and test sets from `dir`. Expects the four standard `.gz` files. |
| `Must` | `func Must(train, test *Dataset, err error) (*Dataset, *Dataset)` | Convenience wrapper — panics on error. |
| `Normalize` | `func Normalize(img []Image) [][]float64` | Scale pixel values from [0, 255] to [0.0, 1.0]. |
| `OneHot` | `func OneHot(label []Label) [][]float64` | Convert labels to 10-element one-hot vectors. |

**Downloading the data**

Run the `mnistdl` make target from the repository root:

```bash
make mnistdl
```

This downloads the four gzip files into `testdata/`:

```
testdata/
  train-images-idx3-ubyte.gz   # 60 000 training images
  train-labels-idx1-ubyte.gz   # 60 000 training labels
  t10k-images-idx3-ubyte.gz    # 10 000 test images
  t10k-labels-idx1-ubyte.gz    # 10 000 test labels
```

**Usage example**

```go
import "github.com/itsubaki/neu/dataset/mnist"

// Load — panics on error (suitable for scripts and examples)
train, test := mnist.Must(mnist.Load("./testdata"))

// Normalize pixel values to [0.0, 1.0]
trainX := mnist.Normalize(train.Image) // [][]float64, shape (60000, 784)
testX  := mnist.Normalize(test.Image)

// One-hot encode labels for use with CrossEntropyError
trainT := mnist.OneHot(train.Label)    // [][]float64, shape (60000, 10)
testT  := mnist.OneHot(test.Label)

// Access raw counts
fmt.Println(train.N) // 60000
fmt.Println(test.N)  // 10000

// Sample a single image and label
img   := trainX[0] // []float64 of length 784
label := train.Label[0]
```

<Note>
The `.gz` files are read directly without decompression to disk — no extra storage is needed beyond the compressed archives.
</Note>

## Penn Treebank (PTB)

The Penn Treebank corpus is a standard benchmark for language modeling. The `ptb` package loads the plain-text version used by many research baselines, tokenizes it at the word level, and builds integer-to-word and word-to-integer mappings.

**Type**

```go
type Dataset struct {
    Corpus   []int            // token IDs for the full text
    IDToWord map[int]string   // integer → word
    WordToID map[string]int   // word → integer
}
```

Newlines in the source text are replaced with the special `<eos>` token before tokenization.

**Functions**

| Function | Signature | Description |
|---|---|---|
| `Load` | `func Load(dir, fileName string) (*Dataset, error)` | Read and tokenize a PTB text file. Use `ptb.TrainTxt`, `ptb.TestTxt`, or `ptb.ValidTxt` as `fileName`. |
| `Must` | `func Must(dataset *Dataset, err error) *Dataset` | Panics on error. |
| `PreProcess` | `func PreProcess(text string) ([]int, map[int]string, map[string]int)` | Tokenize an arbitrary string — useful if you supply your own corpus. |
| `CreateContextsTarget` | `func CreateContextsTarget(corpus []int, windowSize int) ([][]int, []int)` | Slide a window over the corpus to generate CBOW-style context/target pairs. |

**File name constants**

| Constant | Value |
|---|---|
| `ptb.TrainTxt` | `"ptb.train.txt"` |
| `ptb.TestTxt` | `"ptb.test.txt"` |
| `ptb.ValidTxt` | `"ptb.valid.txt"` |

**Downloading the data**

```bash
make ptbdl
```

This downloads three files into `testdata/`:

```
testdata/
  ptb.train.txt
  ptb.test.txt
  ptb.valid.txt
```

**Usage example**

```go
import "github.com/itsubaki/neu/dataset/ptb"

train := ptb.Must(ptb.Load("./testdata", ptb.TrainTxt))

fmt.Println(len(train.Corpus))              // number of tokens in training set
fmt.Println(len(train.WordToID))            // vocabulary size
fmt.Println(train.IDToWord[0])              // first word in vocabulary

// Build context/target pairs for a word2vec or CBOW model
contexts, targets := ptb.CreateContextsTarget(train.Corpus, 2)
// contexts[i] holds the 4 surrounding word IDs; targets[i] is the center word
```

## Arithmetic sequence (addition)

The `sequence` package loads synthetic text datasets where each line is a character-level arithmetic problem and its answer, separated by `_`. The primary file is `addition.txt`, which contains problems of the form `1+10  _  11 ` and is used to demonstrate sequence-to-sequence (seq2seq) models.

**Types**

```go
type Dataset struct {
    Train [][]int  // tokenized question or answer sequences (training split)
    Test  [][]int  // tokenized question or answer sequences (validation split, 10%)
}

type Vocab struct {
    RuneToID map[rune]int
    IDToRune map[int]rune
}
```

`Vocab` translates between characters and integer IDs and provides helpers to reconstruct strings from decoded sequences.

**Functions**

| Function | Signature | Description |
|---|---|---|
| `Load` | `func Load(dir, fileName string, s ...Source) (*Dataset, *Dataset, *Vocab, error)` | Load and tokenize a sequence file. Returns question dataset, answer dataset, shared vocabulary, and error. Shuffles data before splitting. |
| `Must` | `func Must(train, test *Dataset, vocab *Vocab, err error) (*Dataset, *Dataset, *Vocab)` | Panics on error. |

`Load` automatically holds out 10% of the data as a test set, with the remaining 90% as training data.

**File name constants**

| Constant | Value |
|---|---|
| `sequence.AdditionTxt` | `"addition.txt"` |
| `sequence.DateTxt` | `"date.txt"` |

**Downloading the data**

```bash
make additiondl   # addition.txt
make datedl       # date.txt
make datasetdl    # all datasets at once (MNIST + PTB + addition + date)
```

**Usage example**

```go
import "github.com/itsubaki/neu/dataset/sequence"

xData, tData, vocab, err := sequence.Load("./testdata", sequence.AdditionTxt)
if err != nil {
    panic(err)
}

fmt.Println(len(vocab.RuneToID)) // number of unique characters
fmt.Println(len(xData.Train))    // number of training questions
fmt.Println(len(tData.Train))    // number of training answers (same length)

// Decode a predicted sequence back to a string
predicted := []int{3, 7, 2}
chars      := vocab.ToString(predicted) // []string{"1", "2", " "}

// Encode new strings using the vocabulary
ids := vocab.ID([]string{"1+2  "}) // [][]int
```

<Note>
`xData` and `tData` are separate `*Dataset` values holding questions and answers respectively. Index `i` in `xData.Train` corresponds to index `i` in `tData.Train`.
</Note>
