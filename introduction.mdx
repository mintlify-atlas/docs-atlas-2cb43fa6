neu is a deep learning framework written entirely in Go, with no dependencies outside the standard library. It gives Go developers a complete toolkit for building and training neural networks — from simple multilayer perceptrons to sequence-to-sequence models with attention and reinforcement learning agents — without pulling in Python, NumPy, or any C extensions.

## Who it's for

neu is aimed at Go developers who want to build machine learning systems that ship as ordinary Go binaries. If you need reproducible, dependency-free ML inference or training in a Go service, neu lets you stay in the Go ecosystem end-to-end.

<Note>
  neu requires Go 1.22 or later. Because it uses only the standard library, `go get` is all you need — no system libraries, no CGO, no virtual environments.
</Note>

## Architecture overview

neu is organized into focused packages that each own one concern. You can import only the packages you need, and the entire module compiles with `go build` and no additional setup.

## Key capabilities

- **Feedforward networks** — `model.MLP` builds a fully connected network with configurable hidden sizes, batch normalization, and He/Xavier/Glorot weight initialization in a single constructor call.
- **Recurrent models** — `layer.RNN`, `layer.LSTM`, and `layer.GRU` plus their time-distributed counterparts (`layer.TimeRNN`, `layer.TimeLSTM`, `layer.TimeGRU`, `layer.TimeBiLSTM`) cover standard sequence tasks.
- **Seq2seq with attention** — `model.AttentionSeq2Seq` and `model.PeekySeq2Seq` provide encoder-decoder architectures with attention mechanisms for tasks like machine translation and arithmetic sequence learning.
- **Reinforcement learning** — The `agent` package implements tabular and deep RL algorithms. `model.QNet` pairs with `agent.DQN` for value-based deep RL.
- **Model persistence** — `model.Save` and `model.Load` serialize trained parameters to disk using `encoding/gob`, so you can checkpoint and resume training or deploy weights to a separate binary.
