`Sequential` and `MLP` are the two primary feedforward model types in neu. `Sequential` is a generic ordered container that accepts any slice of `Layer` values; `MLP` is a batteries-included multi-layer perceptron built on top of it. Both satisfy the `trainer.Model` interface and can be trained directly with `trainer.Trainer`.

---

## Sequential

`Sequential` stores an ordered slice of layers and a random source. During a forward pass, each layer receives the output of the previous one. The last layer is always the loss layer and is skipped during inference.

```go
type Sequential struct {
    Layer  []Layer
    Source randv2.Source
}
```

<ParamField path="body.Layer" type="[]Layer" required>
  Ordered slice of layers. The final element must be a loss layer (e.g., `*layer.SoftmaxWithLoss`). All preceding layers are run during `Predict`; all layers including the loss are run during `Forward`.
</ParamField>

<ParamField path="body.Source" type="randv2.Source">
  Random source forwarded to layers that need stochasticity at training time (e.g., `BatchNorm`, `Dropout`). If `nil`, behaviour of stochastic layers is undefined.
</ParamField>

### Constructor

```go
func NewSequential(layer []Layer, s randv2.Source) *Sequential
```

Creates a `Sequential` model from an existing layer slice and a random source. The caller is responsible for initialising each layer's weight matrices before passing them in.

### Methods

#### Predict

```go
func (m *Sequential) Predict(x matrix.Matrix, opts ...layer.Opts) matrix.Matrix
```

Runs all layers except the last (loss) layer in forward order. Pass `layer.Opts{Train: false}` for inference-time behaviour (disables dropout, uses running statistics in batch norm).

#### Forward

```go
func (m *Sequential) Forward(x, t matrix.Matrix) matrix.Matrix
```

Runs `Predict` with `Train: true`, then passes the result and the target `t` to the final loss layer. Returns the scalar loss as a `(1, 1)` matrix.

#### Backward

```go
func (m *Sequential) Backward() matrix.Matrix
```

Propagates the upstream gradient `[[1]]` through all layers in reverse order. Must be called immediately after `Forward` because layers cache their intermediate activations. Returns the gradient with respect to the model input (rarely needed in practice).

#### Layers

```go
func (m *Sequential) Layers() []Layer
```

Returns the complete layer slice, including the loss layer.

#### Params

```go
func (m *Sequential) Params() [][]matrix.Matrix
```

Returns all trainable parameters grouped by layer. The outer index corresponds to a layer; the inner index selects a specific matrix in that layer (e.g., `W` at index 0, `B` at index 1 for `Affine`).

#### Grads

```go
func (m *Sequential) Grads() [][]matrix.Matrix
```

Returns gradients with the same shape as `Params()`. Must be called after `Backward` to obtain meaningful values.

#### SetParams

```go
func (m *Sequential) SetParams(p [][]matrix.Matrix)
```

Overwrites layer parameters from `p`. The shape of `p` must exactly match the layout returned by `Params()`.

#### Summary

```go
func (m *Sequential) Summary() []string
```

Returns a string slice with the model type header followed by one `String()` entry per layer. Useful for debugging the layer stack.

## MLP

`MLP` embeds `Sequential` and wires an Affine → BatchNorm → ReLU block for every hidden layer, followed by a final Affine → SoftmaxWithLoss. All configuration comes from `MLPConfig`.

```go
type MLP struct {
    Sequential
}
```

### MLPConfig

```go
type MLPConfig struct {
    InputSize         int
    OutputSize        int
    HiddenSize        []int
    WeightInit        WeightInit
    BatchNormMomentum float64
}
```

<ParamField path="body.InputSize" type="int" required>
  Number of input features per sample.
</ParamField>

<ParamField path="body.OutputSize" type="int" required>
  Number of output classes.
</ParamField>

<ParamField path="body.HiddenSize" type="[]int" required>
  Width of each hidden layer. `[]int{256, 128}` produces two hidden blocks of width 256 and 128. Pass an empty slice for a single-layer network with no hidden units.
</ParamField>

<ParamField path="body.WeightInit" type="WeightInit" required>
  A `func(prevNodeNum int) float64` that returns the weight scale for a given fan-in. Use `weight.He` for ReLU networks, `weight.Xavier` / `weight.Glorot` for sigmoid or tanh, or `weight.Std(s)` for a fixed scale.
</ParamField>

<ParamField path="body.BatchNormMomentum" type="float64">
  Momentum for the running mean and variance in each `BatchNorm` layer. Typical value is `0.9`. A value of `0` disables momentum (running stats are not updated).
</ParamField>

### Constructor

```go
func NewMLP(c *MLPConfig, s ...randv2.Source) *MLP
```

Constructs the full layer stack from config. If no source is provided, `NewMLP` creates a cryptographically seeded source automatically.

### Usage example

```go
import (
    "github.com/itsubaki/neu/model"
    "github.com/itsubaki/neu/optimizer"
    "github.com/itsubaki/neu/trainer"
    "github.com/itsubaki/neu/weight"
)

m := model.NewMLP(&model.MLPConfig{
    InputSize:         784,
    OutputSize:        10,
    HiddenSize:        []int{256, 128},
    WeightInit:        weight.He,
    BatchNormMomentum: 0.9,
})

tr := trainer.New(m, &optimizer.Adam{Alpha: 0.001, Beta1: 0.9, Beta2: 0.999})
tr.Fit(&trainer.Input{
    Train:      xTrain,
    TrainLabel: tTrain,
    Epochs:     20,
    BatchSize:  100,
    Verbose: func(epoch, j int, loss float64, m trainer.Model) {
        // log loss per iteration
    },
})
```

<Note>
  `MLP` overrides `Summary()` to display `*model.MLP` as the type header rather than `*model.Sequential`.
</Note>

## model.Save and model.Load

These package-level functions persist and restore parameter sets using Go's `encoding/gob` format.

```go
func Save(filename string, params [][]matrix.Matrix) error
func Load(filename string) ([][]matrix.Matrix, bool)
```

`Save` creates or truncates the file at `filename` and encodes `params` into it. It returns a non-nil error if the file cannot be created or the encoding fails.

`Load` opens `filename`, decodes the stored parameter slice, and returns it together with a boolean indicating success. It returns `(nil, false)` if the file does not exist or cannot be decoded, so you can safely probe for a checkpoint at startup.

```go
// Persist parameters after training
if err := model.Save("checkpoint.gob", m.Params()); err != nil {
    log.Fatal(err)
}

// Restore parameters before inference
if params, ok := model.Load("checkpoint.gob"); ok {
    m.SetParams(params)
}
```

<Warning>
  The stored parameter layout is tied to the exact model architecture used at save time. Loading a checkpoint into a model with a different layer count or shape will panic or silently corrupt weights.
</Warning>
